# Plan: feature-loop v2.1 — "the loop measures itself"

Make the review loop cheaper per round, crash-safe, and measurable — so an engineering team
can trust it, resume it, and prove it's improving.

## What changes (summary)

- `/mr-review` sends reviewers a **context pack** (conventions + plan decisions) so context-gap
  false positives stop wasting tokens and triage time.
- `/mr-review` reviews **only the delta** on re-review rounds, with prior findings and their
  outcomes attached — no re-litigated stale findings, cheaper rounds.
- `/mr-review` runs **free static tools first** (gitleaks, semgrep, migration lint) so LLM
  tokens go to judgment calls, not pattern matching.
- `/ship-feature` persists **loop state** (`.loop/<slug>.state.json`) at every stage boundary
  and gains `--resume` — an interrupted loop continues instead of restarting.
- The loop **records metrics** (`docs/loops/metrics.jsonl`): rounds to convergence,
  per-reviewer precision, token spend, lead time. New **`/loop-stats`** skill renders trends —
  including whether the blind-spot ledger is actually reducing repeat findings.

## Phases

1. **Reviewer context pack** — `mr-review` Step 2 + `{{CONTEXT}}` in both templates + honesty
   rule (the pack states facts, it never pre-defends the diff).
   Verify: pack builds, capped, disclosed in report. → `feat: add reviewer context pack`
2. **Incremental re-review** — `Reviewed: <sha>` footer on posted briefs; delta mode (diff
   since last reviewed sha + prior findings with outcomes); garbled footer ⇒ full-review
   fallback. Verify: round 2 sends delta only. → `feat: review only the delta on re-review`
3. **Resumable loop state** — `.loop/<slug>.state.json` written at stage boundaries;
   `--resume` + auto-detect; per-stage idempotency contracts.
   Verify: interrupted loop resumes without double-posting. → `feat: persist loop state and add --resume`
4. **Telemetry + `/loop-stats`** — usage capture in `mr-review`; triage tally appended by
   `address-review`; run record by `ship-feature`; new read-only stats skill (scorecard,
   convergence trend, ledger effectiveness). → `feat: record loop metrics per run and triage`,
   `feat: add /loop-stats skill`
5. **Static pre-review** — gitleaks/semgrep if installed + built-in migration lint; one
   comment only if findings; results fed to reviewer context.
   → `feat: run static checks before LLM review`
6. **Docs + release** — README, plugin 2.1.0, personal-copy sync.
   → `docs: document v2.1 features in README`, `chore: bump plugin to 2.1.0`

## Risks
- Footer parsing fragility → missing/garbled footer falls back to full review, never errors.
- Parallel-loop metrics conflicts → acceptable; append-only JSONL merges trivially.
- Token-usage field differences across providers → record what exists, null otherwise.

## Deferred
Plan-review stage, executable acceptance criteria, feature queue + parallel worktrees +
`/loops` dashboard, post-landing watch, ledger→instructions-file promotion.
